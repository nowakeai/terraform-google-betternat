# terraform-google-betternat v0.2.0-alpha.1

Initial GCP alpha module packaging for BetterNAT.

This release wraps the `nowakeai/betternat` provider's
`betternat_gcp_gateway` resource for existing-VPC GCP validation. It targets
Firestore-backed BetterNAT HA, MIG-backed capacity repair, LoxiLB datapath
bootstrap, and optional stable public identity through an existing regional
static external IPv4 address.

## Who Should Use It

Use this release for disposable GCP validation of BetterNAT HA through the
GCP module. Do not use it as a production Cloud NAT replacement without reading
the BetterNAT limitations, IAM, operations, and rollback docs.

## Known Limits

- GCP gateway groups are single-zone.
- Stable public identity uses an existing regional static external IPv4 address
  managed outside the module.
- Connectivity-first handover may temporarily use the target gateway's ordinary
  public IP before the static IP converges back.
- Provider resource updates are replacement-oriented.

## Validation

Before publication, validate the module with:

```sh
terraform fmt -check -recursive
terraform init -backend=false
terraform validate
```

Then run a disposable GCP module smoke through the published provider and record
cleanup evidence.
